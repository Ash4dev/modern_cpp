* Key takeaway
  * Exact sizes are **implementation-dependent**
  * Only minimum guarantees are fixed
  * Always use:
    * `sizeof`
    * `<cstdint>` types (e.g., `int32_t`) when exact size matters

* Fundamental types in C++ are basic built-in types:
  * `void`
  * `std::nullptr_t`
  * Integral types
  * Floating-point types 

  ** String is NOT a fundamental type in C++
    * const char *: string literals are in the .data memory segment
        * read-only (const) & easily move around (pointer)

* `void`: only Functions with no return & Generic pointers
* `std::nullptr_t`: null pointer constant / not a pointer

* Integral types (whole numbers)
  * Basic type: `int`
  * Modifiers:
    * Size → `short`, `long`, `long long`
    * Sign → `signed`, `unsigned`
  * Can be combined (e.g., `unsigned long long int`)

  * Ordering: `char ≤ short ≤ int ≤ long ≤ long long`
    * int →  Numeric Promotion
  * Minimum guarantees: char >= 8 / short >= 16 / int >= 16 / long >= 32 / long long >= 64
      * The keyword int may be omitted if any of the modifiers listed below are used
      * sizeof returns std::size_t

  * The extended integer types are implementation-defined.
  * Note that fixed width integer types are typically aliases of the standard integer types.

  * Integer representation
    * Modern C++ uses **two’s complement**
      * allows positive / negative to be stored with MSB as sign bit
        * simplified ALU desgin, subtract with addition circuits
      * to represent -N: take N, write in binary, invert it, add 1
      * logic: https://www.youtube.com/watch?v=mEsAsRjeqto
        * TODO: https://www.youtube.com/watch?v=z26DA3IAx-Q&t=46s
    * Range for N-bit signed:
      * `-2^(N-1)` to `2^(N-1) - 1`
    * Actual (as opposed to guaranteed minimal) limits available in:
      * C numeric limits interface
      * std::numeric_limits.

  * Special integral types
    * `bool`
    * `char`

* Floating-point types (decimal numbers)
  * `float` → single precision (single machine word used) →  7 decimal places 
  * `double` → double precision (double "") →  Numeric Promotion 
  * `long double` → extended precision (implementation-dependent)

  * Can represent:
    * Infinity (+∞ , -∞ )
    * NaN (not-a-number)
    * Negative zero (-0.0)

  * May have rounding and precision variations

* Data models (affect sizes of types)
  * The choices made by each implementation about the sizes of the fundamental types

  * Four data models found wide acceptance:
    * int / long / pointer (x / y / z)
    * 32-bit systems:
      * LP32 -> Win16 (2 / 4 / 4)
      * ILP32 →  Unix-like (4 / 4 / 4)
    * 64-bit systems:

      * LLP64 →  Win32 = 64-bit (4 / 4 / 8)
      * LP64 →  Unix-like (4 / 8 / 8)
